How to connect plants to the web using Arduino, Processing and Google Docs.

I have always been frustrated with the technical and time requirements needed to view physical sensor data on the web. Services like Xively, ThingSpeak and Nimbits all provide excellent features but they require enough prerequisite knowledge of certain web technologies that may come across as intimidating, especially to those who may be new to programing. While searching for a simpler method to teach to youth, I came across some wonderful examples on journal.code4lib.org and Instructables on how to utilize Google Docs as an easier solution.

I designed this tutorial to give a brief overview of multiple software and hardware components. With that said, this process can be made even simpler with devices such as an Arduino Wifi Shield, Raspberry Pi or an Electric Imp, just to name a few.

Hardware used in this tutorial:

  • Temperature and Humidity Sensor (Grove)
  • Moisture Sensor (Grove)
  • Arduino Uno
  • Breadboard

Software used in this tutorial:

  • Arduino
  • Humidity Temperature Sensor - Arduino Library (How to install)
  • Processing
  • HTTProcessing - Processing Library (How to Install)

1. Arduino

  • Setup your sensors by assembling the breadboard as shown below. In this setup we are only using the breadboard to send power and ground across the side rails. Connect the signal of your humidity & temperature sensor to the Arduino's Digital Input Pin 2 and the signal of your moisture sensor to the Arduino's Analog Input Pin 0

  • Open and run PlantSensors.ino.
    You should now see 3 variables streaming at the bottom of your serial window.

  • According to the manufacturer's wiki here are the value ranges for each sensor:
Sensor Range
Humidity 20% - 90% (RH)
Temperature 0 - 50 (C)
Moisture 0 - 950

2. Google Forms

  • Create a Google Form with 3 text inputs labeled:
    Humidity, Temperature and Moisture

  • While Viewing the Form, copy the url key located here and save it for later, we will need it:

  • Next, save the entry id that is generated for each field by control clicking on the field and selecting Inspect Element (note: this method may vary depending on your browser). Here you can find the entry ID as the string of numbers following #entry_ .

3. Processing

  • Open and run SensorsTest.pde, then check for the streaming variables to make that the data you are receiving from the Arduino is correct.

Depending on your setup, you may need to change your serial input called by changing the variable in the line containing **Serial.list()[1] ** to your Arduino's appropriate number.

  • Next, open Sensors2gDocs.pde and, in the setup of your sketch, replace the XXXXXXXXXX with the information recorded in your google form. Now run the script.

String formURL = "https://docs.google.com/forms/d/**XXXXXXXXXX**/formResponse";
String field1 = "entry.XXXXXXXXXX";
String field2 = "entry.XXXXXXXXXX";
String field3 = "entry.XXXXXXXXXX";

4. Visualize it

  • Use the following formulas to find the last entry in column and add them to the side of your spreadsheet.

=INDEX( A:A ; ROWS( A:A ) )
=INDEX( B:B ; ROWS( B:B ) )
=INDEX( C:C ; ROWS( C:C ) )
=INDEX( D:D ; ROWS( D:D ) )

  • With the Humidity, Temperature and Moisture variables highlighted insert a Gauge Chart.

  • After you have inserted a gauge chart, toggle the menu option in the upper right and select to Publish Chart this will then give you the code needed to embed your gauges in another webpage.

Note: The downside to viewing the gauge in this method is that the gauge chart will only update when the page is reloaded. To have the gauge automatically refresh we will need to call use some javascript to construct how we populate the gauge. Stay tuned for that tutorial to come.
